WEBPACK 사용하기

Page: 홈

  • * WEBPACK

    - 설명: Webpack is a Bundling & Build Orchestaction Tool
    - 왜쓸까?
    ㄴ 1. Code bundles, less imports required
    ㄴ 2. Optimzed code, less code to download (Multiple .ts files & import Http Requst -> additional Network set Time )
    ㄴ 3. More build steps can be added easily

  • * WEBPACK + @

    - webpack: TS를 JS로 변환하고, file bundling작업 등
    - webpack-cli: 프로젝트에서 webpack 명령어를 실행하기 위한 패키지
    - webpack-dev-server: 개발 서버 역할. 파일의 변경사항을 점검하고, 어떤 변경사항이 생겼을 때, 웹페이지를 Re-compile함.
    - ts-loader: webpack에게 어떤 방식으로 Typescript Code를 Javascript code로 compile할 것인지 명세전달

  • * WEBPACK 실행

    - npm package / tsconfig.json / webpack.config.js를 모두 설정한다.
    - package.json에 script를 추가하고 사용한다.

  • * WEBPACK-DEV-SERVER

    - webpack-dev-server에서, bundling된 js파일은,
    컴퓨터 MEMEORY에서만 생성된다.
